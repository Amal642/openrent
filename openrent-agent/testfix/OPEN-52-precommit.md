# OPEN-52 Precommit: Learned Localizer

**Status:** PRECOMMIT (2026-06-05)
**Branch:** docs/project-guide

## Hypothesis

A learned localizer — logistic regression on per-candidate features derived
from past repair outcomes — beats embedding-only top-1 accuracy by ≥15 pp
on ≥20 held-out cross-function cases.

## Claims (independently falsifiable)

### Mechanism claim
Combining structural signals (call-graph membership, file co-location)
with semantic signals (embedding score, BM25 score, name overlap) in a
learned linear model ranks B higher than any single signal alone.

**Falsifier:** LOO-CV on existing 7 cases shows no improvement over embedding
baseline. If Δtop-1 ≤ 0 on the current 7 cases, mechanism is suspect.

### Component claim
The learned localizer beats the embedding-only baseline on the sub-population
it targets: cross-function cases where embedding top-5 does NOT contain B
(i.e., embedding fails, call-graph/structural features carry the load).

**Falsifier:** For cases where B is not in embedding top-5, learned model
does NOT improve rank of B compared to embedding-ranked ordering.

### System claim (§S4 threshold)
On ≥20 held-out cross-function cases, learned localizer top-1 accuracy
≥ embedding top-1 + 15 pp.

**Falsifier:** After generating ≥20 seeds, LOO-CV Δtop-1 < 15 pp.

## Features

Per-candidate feature vector (8 features + bias):
1. `emb_score` — cosine similarity from text-embedding-3-small corpus
2. `bm25_score` — BM25 Okapi score vs query
3. `emb_rank_norm` — 1/emb_rank (0 if not in embedding top-10)
4. `bm25_rank_norm` — 1/bm25_rank (0 if not in BM25 rankings)
5. `in_call_graph` — directly called by entry-point A (binary)
6. `name_in_query` — function name appears in test source+error (binary)
7. `starts_with_underscore` — private helper signal (binary)
8. `same_file_as_entry` — co-location signal (binary)

## Baseline comparison

All baselines are properly featured (not strawmen):
- **embedding**: rank by cosine similarity from 266-fn corpus
- **BM25**: rank by BM25 Okapi score from 266-fn corpus
- **call_graph**: B first if directly called by A (structural oracle; ceiling only)

The learned model must beat the embedding baseline. BM25 is a sanity-check
baseline. Call-graph is ceiling.

## Model

Logistic regression: sigmoid(w · x), L2 regularisation (λ=0.5 on all weights
except bias), trained with scipy.optimize.minimize L-BFGS-B.

Evaluation: leave-one-out CV on all available cases. One model trained on
n-1 cases, evaluated on the held-out case. Repeated for all n cases.

## What does NOT count as success

- Improvement in repair rate without improvement in localization accuracy.
- Top-3 or top-5 improvement without top-1 improvement (repair uses top-1 selection).
- Results on fewer than 20 cases (noise floor too high at n=7).

## Constraint on claims

DO NOT claim "self-learning" or "experience improves the system" until:
1. Δtop-1 ≥ 15 pp AND
2. n ≥ 20 held-out cases AND
3. The held-out cases were generated independently of the training cases
   (no seed contamination).
